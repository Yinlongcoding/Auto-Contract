# YS 单据系统 UI/UX 设计稿 v1

## 设计定位

这套桌面应用本质上是一个外贸单据工作台，使用者会反复录入、核对、导出合同、PI、箱单和发票。界面应该像一套安静、可靠的业务系统：信息密度高，但层级清楚；按钮不花哨，但关键操作一眼能找到。

设计关键词：

- 稳定：固定导航、固定页面头部、内容区域独立滚动。
- 高效：常用字段靠前，金额与箱单/发票计算区可随时看到。
- 可核对：只读、计算、必填、导出状态要有明显区分。
- 可维护：客户、产品、港口、贸易户头等基础资料统一列表体验。

## 信息架构

- 单据生成
  - 合同信息
  - 金额核算
  - 箱单 / 发票信息
  - 导出：合同、PI、PL、CI
- 基础资料
  - 客户管理
  - 客户经理
  - 产品管理
  - 港口管理
  - 贸易户头
  - 条款配置
- 系统配置
  - 模板、编号、默认值、导出路径等后续可归入这里

## 主界面布局

桌面端采用三段式结构：

- 左侧固定导航：宽度约 232px，支持收起。
- 顶部固定页面栏：显示当前模块、说明、主操作按钮。
- 中间内容区：独立滚动，避免导出按钮跟随表单滚走。

单据生成页采用左右分栏：

- 左侧主表单：合同、客户、产品、港口、付款、箱单/发票字段。
- 右侧摘要栏：金额汇总、净重、毛重、尺寸、装桶数量、PO No.。

## 关键交互

- 导出按钮始终位于顶部右侧，避免长表单滚动后找不到操作。
- 表单分组使用清晰标题和小说明，减少用户判断成本。
- 数字输入字段与计算结果视觉分开：可录入字段白底，计算字段浅底。
- 列表页操作栏固定包含：查看、编辑、删除。
- 查看弹窗默认为只读详情，编辑作为明确的二级动作。
- 空联系人或 `/` 在详情和导出逻辑中显示为 `***`，避免生成文档出现脏值。
- 贸易户头 LOGO 不在生成界面额外配置，只跟随贸易户头资料和模板占位图替换。

## 视觉系统

- 主色：深绿色，用于主操作、选中导航、焦点状态。
- 辅助色：琥珀色，用于金额、提示、重要计算结果。
- 背景：冷灰白，保证 Excel/表单类工作场景长时间使用不累。
- 圆角：6px 到 8px，保持业务系统的克制感。
- 字体：优先系统字体，中文界面保证清晰紧凑。

## 需要评审的问题

- 单据生成页右侧摘要栏是否足够好用，还是应改成底部固定汇总条。
- 箱单/发票信息是否需要独立 tab，还是与合同信息放在同一滚动表单更顺手。
- 导出按钮是否保留四个并列按钮，还是改成“导出”下拉菜单。
- 基础资料查看弹窗的信息密度是否合适，是否需要按字段分组。
